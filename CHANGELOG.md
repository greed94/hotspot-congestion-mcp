# Changelog

이 프로젝트는 정식 공개 전까지 `v0.x.y` 버전을 사용합니다.

## 버전 규칙

- `v0.MINOR.PATCH` 형식을 씁니다. 예: `v0.2.0`
- `MINOR`는 사용자가 체감하는 기능 추가, 도구 추가, 응답 구조 개선 때 올립니다.
- `PATCH`는 버그 수정, 문구 수정, 별칭 추가, 배포 설정 변경처럼 기존 기능의 호환성을 유지할 때 올립니다.
- `latest` 이미지는 항상 최신 커밋을 가리키지만, 카카오클라우드 등록에는 가능하면 고정 태그(`v0.2.0` 등)를 씁니다.

## v0.2.0 - 2026-07-03

현재 심사 전 개선 버전입니다.

추가:
- `recommend_less_crowded_hotspots` 도구 추가
  - "지금 갈 만한 한강공원 어디야?"처럼 카테고리 안에서 비교적 한산한 곳을 추천합니다.
  - 지원 카테고리: 전체, 한강공원, 공원, 역세권, 상권, 관광특구, 문화/명소 계열
- `compare_hotspots`를 2곳 고정에서 최대 4곳 비교로 확장
- `best_time_to_go` 응답에 시간대별 날씨 예보 정보 결합
- `list_supported_hotspots`가 서울 주요 121곳 전체를 카테고리별로 반환

개선:
- `홍대입구역 롯데리아`, `뚝섬유원지 근처`처럼 장소명 뒤에 가게명/부가어가 붙어도 지원 권역으로 복구
- 별칭 추가: 남산타워, 더현대, 롯데월드, 뚝섬유원지, 각 한강공원 약칭 등
- 실시간 인구 범위가 비었을 때 `~명`처럼 깨진 문구가 나오지 않도록 가드
- PlayMCP AI가 도구를 더 잘 고르도록 도구 설명 보강
- `CITYDATA`가 없는 비정상 응답은 캐시하지 않도록 안정성 강화

검증:
- 합성 테스트 `test_hotspot.py` 통과
- 실제 서울 API 스모크에서 다중 비교, 한강공원 추천, 날씨 결합, 부가어 매칭 확인
- streamable-http MCP 프로토콜에서 도구 5개 노출 및 추천 도구 호출 확인

## v0.1.0 - 2026-07-03

PlayMCP 임시등록까지 검증한 최초 예선 MVP 기준선입니다.

포함:
- 서울 실시간 도시데이터 `citydata` 실제 API 연동
- 실제 응답 필드 기반 파서
- 서울 주요 121곳 `seoul_areas.json`
- 장소명/별칭 매칭
- 5분 TTL 캐시
- streamable-http MCP 서버
- GHCR 이미지 자동 빌드/푸시 워크플로

도구:
- `get_hotspot_congestion`
- `compare_hotspots`
- `best_time_to_go`
- `list_supported_hotspots`

검증:
- PlayMCP AI 채팅에서 `get_hotspot_congestion` 실제 호출 성공
- 홍대, 홍대입구역, 연남동 실시간 응답 확인
