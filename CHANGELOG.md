# Changelog

`v1.0.0`이 AGENTIC PLAYER 10 예선 제출 기준선입니다.

## 버전 규칙

- `MINOR`는 사용자가 체감하는 기능 추가, 도구 추가, 응답 구조 개선 때 올립니다. 예: 본선 위젯 단계 → `v1.1.0`
- `PATCH`는 버그 수정, 문구 수정, 별칭 추가, 배포 설정 변경처럼 기존 기능의 호환성을 유지할 때 올립니다. 예: `v1.0.1`
- `latest` 이미지는 항상 최신 커밋을 가리키지만, 카카오클라우드 등록에는 가능하면 고정 태그(`v1.0.0` 등)를 씁니다.

## v1.0.2 - 2026-07-09

2차 심사 반려(2026-07-09) 대응. 1차 반려 사유(annotations 부재·서비스명)는 v1.0.1로 통과.

수정:
- (반려 사유) 툴 annotations에 `destructiveHint=False`, `idempotentHint=True` 명시 —
  MCP 스펙상 readOnlyHint=True면 생략 가능하지만 PlayMCP 검사기는 힌트 4종 전부 요구
- (예방) 모든 도구 파라미터에 description 추가 — 미리보기에 null로 노출되던 항목, 3차 반려
  가능성 차단 + AI의 인자 선택 정확도 개선

검증:
- 합성 테스트 15개 통과(annotations 힌트 4종·파라미터 설명 검증 강화)

## v1.0.1 - 2026-07-08

예선 심사 반려(2026-07-08) 대응 + 심사 중 실사용 테스트 140문항에서 확인된 정확성 수정.

수정:
- (반려 사유) 도구 5개 전체에 MCP `ToolAnnotations` 정의 — `title`(서비스명 포함), `readOnlyHint=True`, `openWorldHint`
- (반려 사유) 도구 5개 description에 서비스명 `[핫플 혼잡도 비서]` 포함
- 오타보정(difflib) 단독 후보 자동 확정 제거 → "혹시 '○○' 말씀이신가요?" 확인형으로 변경.
  실서버 테스트에서 미지원 장소가 비슷한 이름의 엉뚱한 데이터로 답하는 사례 확인
  (판교역→교대역, 샤로수길→가로수길). 유사도 수치로는 진짜 오타(강남녁→강남역)와 구분이
  불가해 자동 확정 자체를 없앰. 별칭·정확일치·부분일치·부가어 복구 경로는 영향 없음.
- `recommend_less_crowded_hotspots` 설명에 카테고리 예시 `고궁, 명소` 추가 — AI가 예시에 없는
  카테고리를 몰라 관광특구로 대체 호출하던 문제(테스트 #37) 완화. 라우팅 코드는 원래 정상.

검증:
- 합성 테스트 17개 통과(annotations·서비스명·오탐 차단 테스트 추가)
- 심사 중 실서버 140문항 테스트: 도구 호출 실패·크래시 0건 (별칭·4곳 비교·limit·멀티툴 정상)

## v1.0.0 - 2026-07-07

AGENTIC PLAYER 10 예선 심사 제출 버전입니다. (v0.2.0으로 준비하던 변경분을 포함하며, v0.2.0은 별도 배포되지 않았습니다.)

추가:
- `recommend_less_crowded_hotspots` 도구 추가
  - "지금 갈 만한 한강공원 어디야?"처럼 카테고리 안에서 비교적 한산한 곳을 추천합니다.
  - 지원 카테고리: 전체, 한강공원, 공원, 역세권, 상권, 관광특구, 문화/명소 계열
  - 상한 없이 카테고리 전체를 스캔합니다(`전체` 선택 시 121곳 모두).
- 백그라운드 캐시 워머
  - 서버가 5분 주기로 121곳을 미리 갱신해, 추천·조회가 서울 API를 기다리지 않고 즉시 응답합니다.
  - 콜드 상태 recommend(전체) 38초 → 워머 적용 후 0.001초.
- `compare_hotspots`를 2곳 고정에서 최대 4곳 비교로 확장
- `best_time_to_go` 응답에 시간대별 날씨 예보 정보 결합
- `list_supported_hotspots`가 서울 주요 121곳 전체를 카테고리별로 반환

개선:
- 모든 응답에 데이터 기준시각을 표기(예: `(2026-07-06 16:25 기준)`). 서울 인구 데이터는 5분 주기로 갱신되고 제공까지 약 30분 지연되므로, "지금" 대신 실제 관측 시각을 노출합니다.
- `홍대입구역 롯데리아`, `뚝섬유원지 근처`처럼 장소명 뒤에 가게명/부가어가 붙어도 지원 권역으로 복구
- 별칭 추가: 남산타워, 더현대, 롯데월드, 뚝섬유원지, 각 한강공원 약칭 등
- 실시간 인구 범위가 비었을 때 `~명`처럼 깨진 문구가 나오지 않도록 가드
- PlayMCP AI가 도구를 더 잘 고르도록 도구 설명 보강
- `CITYDATA`가 없는 비정상 응답은 캐시하지 않도록 안정성 강화
- 모든 응답 하단에 데이터 출처(서울시 실시간 도시데이터) 명시
- 서울 API 일시 장애 시 직전 성공 데이터로 응답(스테일 폴백, 기준시각 표기 유지)
- 12시간 예측이 전부 붐빌 때 '한산한 시간대' 대신 '그나마 나은 시간대'로 정직하게 안내

검증:
- 합성 테스트 `test_hotspot.py` 통과
- 실제 서울 API 스모크에서 다중 비교, 한강공원 추천, 날씨 결합, 부가어 매칭 확인
- 캐시 워머로 recommend(전체 121곳) 응답 0.001초 확인(콜드 38초 → 웜 즉답)
- 서울 API 실측: 1시간 4,840콜 무오류(일반 인증키 일일 호출 한도 없음), 인구 5분 주기 갱신 확인
- streamable-http MCP 프로토콜에서 도구 5개 노출 및 추천 도구 호출 확인
